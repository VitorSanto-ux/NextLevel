O MVC que fiz é pra ser facil de entender e claro de ver, de mexer. O MVC tem os seguintes componentes (Model-Views-Controller). O Model é responsável pela manipulação e gerenciamento dos dados, realizando a comunicação direta com os bancos de dados. O Views é responsável pela apresentração dos dados ao usuário. O Controller recebe as requesições do usuário, e processa as regras. Aciona o Model para a manipulação dos dados, e retornando as informações adequedas para serem exibidas.
A classe EventoModel, responsável pela gestão dos dados relacionados aos eventos, incluindo operações de cadastro, consulta, atualização e exclusão. A classe ParticipanteModel tem a função de gerenciar as informações dos participantes, permitindo seu cadastro e listagem. Por sua vez, a classe InscricaoModel é responsável por controlar o vínculo entre participantes e eventos, garantindo o registro adequado das inscrições e o controle da quantidade de participantes.
A classe EventoController gerencia as requisições relacionadas aos eventos, recebendo dados provenientes das interfaces e acionando o Model correspondente. A classe ParticipanteController é responsável pelo processamento das ações relacionadas aos participantes, enquanto a classe InscricaoController controla o fluxo de inscrições, incluindo validações e restrições, como o limite máximo de participantes por evento.
